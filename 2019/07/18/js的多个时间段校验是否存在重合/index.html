js的多个时间段校验是否存在重合

  • js

实现思路

在设置多个时间段时需要实时校验这些设置的时间段是否存在重合，校验重合的思想是：

  • 首先将这些时间段按照开始时间的升序排列
  • 其次，对这些排序过后的数组对象进行循环遍历，比较当前时间对象的结束时间是否大于下一个时间对象的开始时间，如果大于，则表明时间存在重合，就将这两项return出去，结束循环。
  • 最后，实时监听这个数组的长度变化，如果数组长度存在，则表明存在重合项，否则不存在重合项。

卡壳地方

  • 对这个时间对象有重合项不知道从哪儿着手；
  • 比较的时候没有对该数组进行排序，直接用遍历的数组对象中的当前时间对象与其他时间对象进行比较，该项的开始时间和结束时间是否在其他项的开始结束时间中间，如果有就保存下来，利用了两层循环，导致无法比较出上一项的结束时间与下一项的开始时间相同的时候也校验为重复项，代码逻辑不清晰；
  • 利用两层循环不仅耗性能，而且在比较时也不能准确判断出是否真的存在重复项，排查错误不好处理。
